Getting Started

Think about that you wish to build a table with following Columns:
"Product Name", "Unit Price", "Buying Price", "Sold Count".
Also, you have a REST api call product_info, and it provides JSON data in following format: 

[
    {   
        "id": 1
        "name": "Cloth A",
        "unit_price": "10.00",
        "buying_price": "5.00",
        "sold_count": 30
    },
    {
        "id": 2
        "name": "Cloth B",
        "unit_price": "15.00",
        "buying_price": "8.00",
        "sold_count": 120
    },
    {
        "id": 3,
        "name": "Shoe C",
        "unit_price": "50.00",
        "buying_price": "35.00",
        "sold_count ": 47.
    },
    ...,
]

What you can do is: either do a few coding to generate the table, or use Trackable Table with some easy configuration. Please be noticed, only the "text" field is mandatory, it is smart enough to fall back to the default parameters if they are not provided in the configuration.

Code: Preparation-Part 

widget.init({
    name:   {text: 'Product Name', // text in th
            changeable:true,    // user can change it
            width: '55%'},  // width

    unit_price: {
            text: 'Unit Price', 
            tpl: '${{val}}',   // how would you like to display the value in td.
            width:'15%'},

    buying_price: { 
            text: 'Buying Price',
            tpl: '${{val}}',
            changeable: true,
            width:'15%',
            rule:/^\d+[\.]{1}\d{2}$/,  // rule to verify the user input
            tips: 'Please input number with two digits after decimal point. e.g. 23.58.'}, 
            // pop up tips when the verification is not passing

    sold_count: {
        text: 'Sold Count',
        changeable:true,
        width:'15%',
        rule:/^\d+$/,
        tips: 'Please input integer only. e.g. 15.'}
    },

    'product_info' // REST-api url to get the table data.
);

// append the element to where you wish to display;
$('div.dt_placeholder').append(
    widget.layout_view.$el);

        // this will load up the data from the url you have configure.
